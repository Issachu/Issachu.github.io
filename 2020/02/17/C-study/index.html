编程机制

C语言属于可移植性语言，可以在不同的环境中使用(包括5种环境：UNIX、Linux、Windows、Mac OS、MS-DOS)。
用C语言编写程序时，大部分编写内容被储存在文本文件（源代码文件；source code file）中，文件名以 .c为结尾。（在文件名中，点号‘.’前面的是基本名，后面为扩展名）
C程序的基本策略是用程序通过编译和链接把源代码转化成可执行文件，编译器把源代码转换成中间代码，链接器把中间代码和其他代码合并，生成可执行文件；链接器还能将程序和预编译的库代码合并。
中间文件把源代码转换为机器语言代码，结果储存在目标代码文件。目标代码文件缺失启动代码（startup code），启动代码作为程序和操作系统的接口，不同的操作系统对应不同的启动代码。目标代码还缺少库函数。
总结：源代码>编译器>目标代码>链接器（库代码+启动代码）>可执行代码
